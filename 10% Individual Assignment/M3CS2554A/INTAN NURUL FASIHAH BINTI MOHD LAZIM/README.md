## INTRODUCTION
A Python-based program called the Parallel Dice Combionation Simulator rolling two dice a very large number of times, such as 1,000,000 rolls. This project aims to illustrate the differences between parallel programming (multiprocessing) and concurrent programming (threading) in managing computational activities. 

The software determines the frequency of each conceivable dice total (ranging from 2 to 12) and contrasts the two methods' execution times.

The project's highlights are:

- Effienciency CPU use
- Effienecy enhancement
- Practical application of parallel computing

  ------------
 ## OBJECTIVE
 - To replicate large-scale combination of dice rolls
 - To implement threading for concurrent execution
 - To put multiprocessing into practice for concurrent execution
 - To evaluate how well the two methods perform
 - To examine dice sums probability distribution
